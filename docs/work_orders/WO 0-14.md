# WO3 — Combat Engine

### Modules

M5 Combat Engine

M6 Battle Contract

### Purpose

Simulate deterministic 1v1 combat.

### Location

```
apps/backend/src/battle/
```

### Core Rules

Round limit

```
30 rounds
```

Initiative

```
initiative += SPD
while initiative >= 100:
    act
    initiative -= 100
```

Actor ordering

```
initiative DESC
SPD DESC
entityId ASC
```

### Combat Calculations

Hit chance

```
hitChanceBP = clamp(A - E + skillAcc, 500, 9500)
rollBP = rng.int(1..10000)
hit = rollBP <= hitChanceBP
```

Damage

```
damage = floor((basePower + ATK) * 100 / (100 + DEF))
```

### Status System

Duration

```
1–3 turns
```

Stacking

```
NOT allowed
```

Reapply

```
refresh duration
```

### Cooldowns

```
turn-based
```

### Output

```
BattleResult
```

Including ordered `BattleEvent[]`.

### Acceptance Tests

- deterministic logs
- initiative ordering correct
- timeout at round 30
- status refresh works
- cooldown decrement correct

# WO4 — AI Decisioning

### Modules

M8 AI Decisioning

### Purpose

Deterministic decision system selecting from:

```
basic attack
active skill 1
active skill 2
```

### Inputs

```
self HP%
enemy HP%
statuses
cooldowns
enemy archetype
weakness tags
learned weights
```

### Decision Method

Score actions:

```
score = base
      + matchup bonuses
      + timing bonuses
      - waste penalties
      + learned weight
```

Highest score chosen.

Tie-break

```
skillId ASC
basic last
```

### Acceptance Tests

- identical inputs produce identical action
- learned weight biases decisions
- cooldown respected

# WO5 — Learning Update

### Modules

M10 Learning Persistence

### Purpose

Update skill effectiveness weights after battle.

### Table

```
character_skill_weights
```

Fields

```
characterId
archetypeId
skillId
weight_int
```

### Update Algorithm

Contribution

```
damagePart = damageDealt * 1000 / enemyHPMax
statusPart = statusTurnsApplied * 1000 / 3
contrib = (700*damagePart + 300*statusPart)/1000
```

Update

```
delta = sign * learningRate * contrib / 1000
```

Clamp

```
[-1000, +1000]
```

### Acceptance Criteria

- weights update only after battle
- bounded values
- deterministic updates
